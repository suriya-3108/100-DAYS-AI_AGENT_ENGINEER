## 📚 What I Learned

| Concept                  | Explanation                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| **Stack**                | A data structure that follows **Last-In-First-Out (LIFO)**                             |
| **Push**                 | Add an element to the top of the stack                                                 |
| **Pop**                  | Remove the element from the top of the stack                                           |
| **Peek / Top**           | View the top element without removing it                                               |
| **isEmpty**              | Check if the stack has no elements                                                     |
| **Balanced Parentheses** | Use stack to track opening brackets and ensure each closing bracket matches            |
| **Postfix Expression**   | Mathematical expression where operators come **after operands**; evaluated using stack |
| **Reverse String**       | Push each character to stack and pop all to reverse the string                         |

---

## 💡 Key Takeaways

| Takeaway                      | Why It Matters                                              |
| ----------------------------- | ----------------------------------------------------------- |
| **Stacks are LIFO**           | Useful in parsing, undo-redo, recursion, and backtracking   |
| **Push and pop are the core** | All stack operations depend on adding/removing top elements |
| **Balanced parentheses**      | Stack helps track nesting and matching brackets             |
| **Postfix evaluation**        | Stack allows efficient computation of expressions           |
| **Reversing a string**        | Shows real-world application of LIFO behavior               |

---

## 🔍 Challenges & Solutions

| Challenge                              | Solution                                                         |
| -------------------------------------- | ---------------------------------------------------------------- |
| Understanding operand order in postfix | Remember: first pop = second operand, second pop = first operand |
| Empty stack on closing bracket         | Check stack before popping to avoid errors and detect imbalance  |
| Reversing a string correctly           | Pop all characters sequentially to get reversed string           |
